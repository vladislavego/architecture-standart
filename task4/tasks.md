# Task4 — Список крупных задач для будущего планирования

Документ содержит перечень крупных задач для каждой системы,
сформированный на основе архитектуры MVP открытия депозитов (Task3)
и её расширения для передачи ставок в кол-центры (Task4).

Новые задачи не вводят дополнительных архитектурных компонентов,
а развивают и переиспользуют решения MVP.

---

## АБС

1. Формализация модели ставок в АБС  
   (единое представление ставок как источника истины)

2. Расширение адаптера интеграции АБС  
   для передачи актуальных ставок во внешний контур

3. Реализация передачи ставок в Deposit Applications Service  
   (по расписанию или по управляемому событию)

4. Обеспечение контроля целостности и актуальности данных ставок

5. Логирование операций экспорта и передачи ставок

**Комментарий:**  
АБС остаётся источником истины. Прямой онлайн-доступ из кол-центров исключён.

---

## Deposit Applications Service

1. Расширение сервиса для хранения и обработки данных по ставкам

2. Получение ставок из АБС через существующий адаптер интеграции

3. Предоставление актуальных ставок системе кол-центра банка
   через управляемый интерфейс

4. Реализация механизма формирования файлов со ставками
   для партнёрского кол-центра

5. Хранение версий ставок и временных меток обновления

6. Логирование и мониторинг процессов обновления и распространения ставок

**Комментарий:**  
Сервис переиспользуется и расширяется. Новые ключевые сервисы не вводятся.

---

## Система кол-центра банка

(Java Spring Boot + React + PostgreSQL)

1. Интеграция с Deposit Applications Service для получения ставок

2. Реализация пользовательского интерфейса
   для отображения актуальных ставок операторам

3. Обновление рабочих скриптов операторов
   с учётом нового процесса консультаций

4. Обучение операторов работе с обновлённым функционалом

5. Обработка сценариев недоступности ставок
   (fallback и уведомления)

**Комментарий:**  
Интеграция реализуется без прямого доступа к АБС.

---

## Партнёрский кол-центр

1. Согласование формата файла со ставками
   (структура, кодировки, временные метки)

2. Настройка процесса приёма файлов на стороне партнёра

3. Обновление скриптов операторов партнёрского кол-центра

4. Организация процесса обновления ставок по расписанию

5. Контроль получения и актуальности файлов со ставками

**Комментарий:**  
Интеграция осуществляется исключительно файловым способом.
API-взаимодействие не используется.

---

## Интеграция и IT-инфраструктура банка

1. Реализация механизма файловой передачи ставок
   партнёрскому кол-центру

2. Настройка расписаний обновления и регламентов передачи данных

3. Обеспечение мониторинга и алертинга процессов передачи ставок

4. Настройка логирования и хранения артефактов обмена

5. Подготовка технической и эксплуатационной документации

**Комментарий:**  
Задачи обеспечивают надёжность, воспроизводимость и поддерживаемость решения.

---

## Итог

Сформированный перечень задач:

- полностью опирается на архитектуру MVP;
- не вводит новых ключевых компонентов;
- используется как основа для построения RoadMap на горизонте 6 месяцев;
- закладывает основу для дальнейшей автоматизации на горизонте года.
